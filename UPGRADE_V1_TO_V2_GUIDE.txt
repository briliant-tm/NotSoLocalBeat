LOCALBEAT UPGRADE GUIDE: V1 → V2.0
================================================================================

Comprehensive guide to changes from LocalBeat V1 (Singleplayer) to V2.0

================================================================================
WHAT'S NEW IN V2.0
================================================================================

MAJOR FEATURES ADDED:

1. MULTIPLAYER GAMEPLAY
   - Support for 2-6 players per room
   - Host creates room with customizable settings
   - Players join with 6-character room codes
   - Real-time synchronization via polling
   - Independent score tracking
   - Ready system (all players must confirm before start)

2. AUTHENTICATION SYSTEM
   - User registration with email (optional)
   - Secure login with bcrypt password hashing
   - Guest mode (play without registration)
   - Session management with cookies
   - User profile tracking

3. YOUTUBE PLAYLIST SUPPORT
   - Connect YouTube playlists directly
   - Random song selection from playlists
   - Configurable start timestamp (0-80% of duration)
   - Works in both singleplayer and multiplayer
   - Maximum 50 songs per playlist (performance)

4. IMPROVED UI/UX
   - Clean authentication interface (no emoji)
   - Mode selector (Singleplayer vs Multiplayer)
   - Game configuration pages
   - Real-time player list display
   - Dynamic spectrum visualization
   - Multi-language support (English & Indonesian)

5. DATABASE SYSTEM
   - SQLAlchemy ORM for data persistence
   - 7 database models for complete data tracking
   - Query caching for performance
   - PostgreSQL support for production

6. COMPREHENSIVE DOCUMENTATION
   - Gameplay tutorial (300+ lines)
   - Developer documentation (600+ lines)
   - Deployment guide for Vercel
   - Quick start guide for local testing
   - Implementation summary

ARCHITECTURE CHANGES
================================================================================

V1 ARCHITECTURE:
- Monolithic Flask app (single file)
- File-based storage (no database)
- Single-player only
- Local music only
- No authentication
- No real-time features

V2.0 ARCHITECTURE:
- Modular Flask app + models
- SQLite (dev) / PostgreSQL (prod)
- Single + Multiplayer
- Local + YouTube music
- Full authentication system
- Real-time multiplayer sync
- Environment-based configuration

STRUCTURAL COMPARISON:

V1 App Flow:
    Start → Choose Mode → Play Game → Results

V2.0 App Flow:
    Start → Auth → Choose Mode → Config → Game → Results
            (3 paths: Register, Login, Guest)

DATABASE CHANGES
================================================================================

V1 (NO DATABASE):
- Music list: Scanned from file system
- Game history: JSON file (game_history.json)
- User data: Not tracked
- Score storage: Session memory only
- Limitations: No persistence, single device only

V2.0 (SQL DATABASE):
- Users: User registration and authentication
- Rooms: Multiplayer room management
- Sessions: Game session tracking
- Scores: Detailed score per question
- Cache: Audio analysis results
- History: Play history for smart recommendations

NEW TABLES:
1. users (id, username, email, password_hash, is_guest, created_at)
2. game_rooms (id, room_code, host_id, mode, max_players, ...)
3. room_players (id, room_id, user_id, is_ready, is_host, ...)
4. game_sessions (id, room_id, user_id, current_question, ...)
5. player_scores (id, session_id, question_number, score, ...)
6. question_cache (id, file_path, drop_time, duration, ...)
7. play_history (id, user_id, song_name, played_at)

BENEFITS:
- Multi-user support
- Data persistence
- Analytics ready
- Scaling ready
- Cloud deployment ready

UPGRADE PATH
================================================================================

MIGRATING FROM V1 TO V2.0:

1. BACKUP OLD DATA:
   - Save music files (still used)
   - Backup game_history.json if needed
   - Save any customization

2. INSTALL NEW DEPENDENCIES:
   pip install -r requirements.txt

3. CONFIGURE ENVIRONMENT:
   - Copy .env.example to .env
   - Set MUSIC_FOLDER path
   - Set SECRET_KEY
   - (For production: Set DATABASE_URL)

4. DATABASE SETUP:
   - Database auto-creates on first run
   - No migration needed

5. TEST:
   - Register account
   - Play singleplayer
   - Test multiplayer (2 windows)
   - See QUICK_START_GUIDE.txt

6. DEPLOY:
   - For Vercel, follow VERCEL_DEPLOYMENT_GUIDE.txt
   - Set environment variables
   - Push to GitHub
   - Deploy via Vercel

BREAKING CHANGES:
- OLD game_history.json not imported
- Old routes different
- New database required
- Authentication required (except guest)

NON-BREAKING:

FAQ: V1 TO V2.0
================================================================================

Q: Will my old music files still work?
A: Yes, all supported formats (MP3, FLAC, WAV, OGG, M4A) still work.

Q: Can I disable multiplayer and just use singleplayer?
A: Yes, multiplayer is optional. Click Singleplayer from mode selector.

Q: Do I have to register an account?
A: No, guest mode available without registration.

Q: Can I use YouTube playlists instead of local music?
A: Yes, toggle YouTube option in host setup.

Q: Will Vercel deployment work with my music folder?
A: No, local filesystem not available on serverless. Use YouTube mode instead.

Q: How many players can join a room?
A: 2, 4, or 6 players (configurable by host).

Q: Is there a database migration needed?
A: No, database auto-creates on first run.

Q: Can I go back to V1?
A: V1 code is in app_backup.py if needed, but recommended staying on V2.0.

Q: How is performance compared to V1?
A: First game slower (audio analysis), subsequent games faster (caching).

Q: Are there any breaking changes?
A: Old game_history.json not imported. Need new accounts. Otherwise compatible.
