================================================================================
POTENTIAL ISSUES TO MONITOR
================================================================================

1. LIBROSA PERFORMANCE
   - Audio analysis might be slow for large files
   - Solution: Already implemented with caching
   - Priority: Monitor performance, adjust cache expiry if needed

2. YOUTUBE RATE LIMITING
   - yt-dlp might hit rate limits on high usage
   - Solution: Implement playlist caching, limit playlist size to 50
   - Priority: Monitor in production

3. DATABASE CONNECTIONS ON SERVERLESS
   - SQLite won't work on Vercel (no persistent filesystem)
   - Solution: Use PostgreSQL for production
   - Priority: Must use PostgreSQL for Vercel deployment

4. WEBSOCKET UNAVAILABLE
   - Vercel doesn't support WebSocket
   - Solution: Using polling (1s interval) - works but not optimal
   - Priority: Consider migration to service with WebSocket support

5. FILE SIZE LIMITS
   - Large music files might exceed timeout
   - Solution: Implement file size checking, stream instead of load
   - Priority: Add validation for audio file size

6. BROWSER COMPATIBILITY
   - Web Audio API not available in old browsers
   - Solution: Already using feature detection
   - Priority: Test in Safari and Edge

================================================================================
KNOWN LIMITATIONS
================================================================================

1. VERCEL SERVERLESS
   - No local file system access (music folder scanning won't work)
   - Max 60 second function runtime
   - PostgreSQL required (not SQLite)

2. YOUTUBE INTEGRATION
   - Cannot directly stream YouTube audio to browser (CORS/privacy)
   - Playlist limited to 50 songs (performance)
   - Requires yt-dlp or similar library

3. MULTIPLAYER
   - Max 6 players per room (design choice)
   - Polling-based (1s latency)
   - No real-time chat

4. AUDIO PROCESSING
   - Librosa requires significant CPU (can timeout on serverless)
   - FFmpeg needed for audio extraction
   - Limited audio format support on some systems

5. BROWSER CONSTRAINTS
   - Web Audio API not available in IE
   - CORS issues with some CDNs
   - WebSocket not available (using polling instead)
